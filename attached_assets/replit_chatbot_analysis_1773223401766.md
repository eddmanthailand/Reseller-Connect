# รายงานวิเคราะห์การทำงานและปัญหาของ Chatbot (สำหรับ Replit/ทีมพัฒนา)

จากการตรวจสอบโค้ดในโปรเจกต์อ้างอิง `https://github.com/eddmanthailand/Reseller-Connect` (เฉพาะส่วน `app.py` ฟังก์ชัน `_bot_chat_reply`) เพื่อหาสาเหตุที่บอทสูญเสียบริบทการสนทนาและตอบคำถามผิดพลาด เช่น ตอบเป็น "กระโปรงทรง A" เมื่อลูกค้าถามเรื่อง "ชุดพยาบาล" พบว่าปัญหาเกิดจาก **Logic ในส่วน Backend ซ้อนทับการตัดสินใจของ AI และกฎ Prompt ที่เข้มงวดเกินไป**

## 🛑 จุดอ่อนและปัญหาที่พบในโค้ด

### 1. การล้างบริบท (Context Reset) ที่รุนแรงเกินไปจากการใช้ N-gram Search
(อ้างอิง `app.py` ประมาณบรรทัดที่ 16141-16179)
โค้ดมีลอจิกในการตรวจจับการเปลี่ยนเรื่อง (Context-switch detection) โดยนำข้อความล่าสุดของลูกค้ามาหั่นเป็นคำสั้นๆ แบบ N-gram (ความยาว 4, 5, 6 ตัวอักษร) แล้วนำไปค้นหา 
* **สิ่งที่เกิดขึ้น:** เมื่อลูกค้าถามคำถามต่อเนื่อง เช่น "ถ้าสัดส่วน อก 34 นิ้ว എว 28 นิ้ว..." คำเหล่านี้ไม่ได้รวมอยู่ใน "ชื่อสินค้าเดิม (ชุดพยาบาล)" แต่เผอิญไปตรงกับคำอธิบาย (Description) ของสินค้าอื่น (เช่น กระโปรงทรง A ที่อาจมีคำว่า เอว หรือ 28)
* **ผลกระทบ:** ระบบ Backend สั่งล้างข้อมูล `current_product_id = None` ทันที ทำให้บอทลืมว่าลูกค้ากำลังพูดถึงชุดพยาบาลอยู่ และโหลดข้อมูลสินค้ายัดเข้าไปใหม่เป็น "กระโปรงทรง A" แทน

### 2. Prompt Rule ห้ามใช้ข้อมูลจากประวัติแชท (Strict Instruction)
(อ้างอิง `app.py` บรรทัดที่ 16517)
ใน System Prompt มีการตั้งกฎอย่างเด็ดขาดว่า:
> `❌ ห้ามใช้ประวัติแชทเดิมเป็นข้อมูลสินค้า ต้องอ้างอิงจากรายการสินค้าด้านล่างเท่านั้น`
* **ผลกระทบ:** แม้ Gemini LLM จะอ่านประวัติแชทและรู้ตัวว่าลูกค้ากำลังถามถึง "ชุดพยาบาล" ในกล่องข้อความที่แล้ว แต่อัลกอริทึม N-gram ได้บังคับยัด "กระโปรงทรง A" เข้ามาในรายการสินค้าด้านล่าง LLM จึงต้องทำตามคำสั่งอย่างเคร่งครัดโดยทิ้งบริบทเดิม แล้วตอบตามรายการสินค้าที่ถูกยัดเข้ามาใหม่ จึงเกิดเป็นประโยคประหลาดที่ว่า *"คุณพี่แจ้งรอบอก... จากรูปตารางไซส์ **กระโปรงทรง A**"*

### 3. การค้นหา Keyword ไม่เจอ ทำให้ Products Text ว่างเปล่า
เมื่อลูกค้าถามเจาะจงที่ไม่ได้กี่ยวกับชื่อสินค้า เช่น "เนื้อผ้าเป็นแบบไหน ยืดหยุ่นไหม" ลอจิกการค้นหา (Keyword/N-gram search) จะไม่พบสินค้าใดๆ ในฐานข้อมูลที่มีชื่อตรงกับคำนี้ ทำให้ตัวแปร `products_text` กลายเป็นค่าว่าง บอทจึงไม่มีข้อมูลเนื้อผ้าไปตอบ และเข้าสู่ Fallback ตอบว่า "ในระบบยังไม่มีข้อมูลเกี่ยวกับเนื้อผ้า..." (ตามที่โค้ดสั่งให้ Prompt เลี่ยงการตอบมั่วเมื่อไม่มีข้อมูล)

## 🛠️ คำแนะนำในการแก้ไข (Recommendations for Replit)

### 1. ยกเลิกการใช้ N-gram Search ในการสลับ Context (Let AI Handle Context)
ปัญหาเกิดจากการให้ Python (Backend) พยายามเอาชนะการอ่านบริบทของ AI ด้วยการเดา Keywords **ควรยกเลิกการบังคับเคลียร์ `current_product_id` ด้วย N-gram Search** 
* **ทางแก้:** ปล่อยให้ Gemini AI ซึ่งอ่าน `history_text` อยู่แล้วเป็นคนประเมินเองว่า "ตอนนี้กำลังเปลี่ยนผลิตภัณฑ์หรือไม่" 
* ระบบปัจจุบันมีการรับค่ากลับเป็น JSON (`new_state.current_product_id`) อยู่แล้ว ควรให้ AI เป็นฝ่ายสั่งการเปลี่ยน state นี้เองตามบทสนทนา แทนที่จะให้ Backend ตัดสินใจตัดบท

### 2. ปรับปรุง Prompt ให้ยืดหยุ่นต่อการใช้ประวัติแชท
* **ทางแก้:** เปลี่ยนการเขียน Prompt ห้ามเด็ดขาดให้เป็นในเชิงแนะนำ เช่น:
> `"✅ สังเกตจากประวัติแชทว่าลูกค้ากำลังพูดถึงสินค้าใดอยู่ หากเป็นการถามต่อเนื่องให้คงบริบทสินค้านั้นไว้ แต่รายละเอียดของสต็อก/ราคา/สเปค ต้องอ้างอิงจากรายการข้อมูลด้านล่างเป็นหลัก ห้ามแต่งขึ้นมาเอง"`

### 3. เมื่อมี Context อยู่แล้ว ไม่ควรกระตุ้นระบบค้นหาซ้ำซ้อน
ถ้า `current_product_id` มีค่าใน Session Data อยู่แล้ว ควรโฟกัสแค่การ Query ข้อมูลของ Product ID นั้นขึ้นมาป้อนให้ Prompt แบบละเอียดก็พอ ไม่จำเป็นต้องวิ่งรอบค้นหาด้วย Keyword ใหม่ในทุกๆ พิมพ์ เว้นเสียแต่ว่าผู้ใช้พิมพ์ชัดเจนว่าอยากดูสินค้าอื่นๆ

### 4. ดึงข้อมูลเนื้อผ้า (Fabric Info) ให้ครบถ้วน
ถ้าลูกค้ามักจะถามเรื่องเนื้อผ้า ควรให้แน่ใจว่าใน Query ของ Database (`_product_base_select`) มีการดึงคอลัมน์ `product.description` หรือคุณสมบัติลงมาด้วยเสมอ (ปัจจุบันดึงแค่ `bot_description` เป็นหลัก) เพิ่อให้ LLM อ่านและหยิบไปตอบคำถามย่อยได้
